FASE 1 PASSO 00 DE 11
Fundação do PS42 Luanda

Esta fase estabelece toda a base técnica do projecto. Ao final, terás o Next.js a correr, login via 42 Intra funcional, a base de dados estruturada e o design system base pronto.

Duração estimada: 2–3 horas
O que vamos construir nesta fase
⚡
Projecto Next.js 14
App Router, TypeScript, Tailwind CSS. Estrutura de pastas completa e organizada para crescer até produção.
🔐
Login com 42 Intra
OAuth2 via NextAuth.js. Qualquer cadete ou staff entra com a conta da 42 — sem passwords separadas.
🗄️
Base de Dados
PostgreSQL no Supabase com Prisma ORM. Schema das 12 tabelas criado e migrado.
🎨
Design System Base
Cores, tipografia, componentes base. shadcn/ui configurado. Layout de cadete e staff separados.

💡 Route Groups
As pastas entre parênteses (cadete) e (staff) são "Route Groups" do Next.js. Agrupam rotas sem afectar a URL — o URL de /dashboard não terá /cadete/ no caminho.

DATABASE PASSO 05 DE 11
Supabase + Prisma

Conectar o Prisma ao Supabase e criar o Prisma Client singleton — a porta de entrada de toda a comunicação com a base de dados.

~20 minutos
5.1 — Obter as URLs do Supabase

No dashboard do Supabase, navega para o teu projecto:

1. Entra em supabase.com → abre o teu projecto
2. Vai a Project Settings → Database
3. Em Connection string, selecciona URI
4. Copia a string com ?pgbouncer=true → é o DATABASE_URL
5. Copia a string sem pgbouncer (porta 5432) → é o DIRECT_URL
5.2 — Prisma Client Singleton

Este padrão é essencial no Next.js — evita criar múltiplas instâncias do Prisma em desenvolvimento:

lib/prisma.ts TYPESCRIPT
import { PrismaClient } from '@prisma/client' const globalForPrisma = globalThis as unknown as { prisma: PrismaClient | undefined } export const prisma = globalForPrisma.prisma ?? new PrismaClient({ log: process.env.NODE_ENV === 'development' ? ['query', 'error', 'warn'] : ['error'], }) if (process.env.NODE_ENV !== 'production') globalForPrisma.prisma = prisma
5.3 — Testar a conexão com o Supabase
TERMINAL
npx prisma db pull # Se a conexão funcionar, vês algo como: # ✔ Introspected 0 models and wrote them into prisma/schema.prisma # (0 modelos porque a DB está vazia — é normal)
💡 Se der erro de conexão
Verifica se as URLs do Supabase estão correctas no .env.local e se substituíste [password] pela tua password real do projecto Supabase (Settings → Database → Database password).
✅ Checklist do Passo 05
  • DATABASE_URL e DIRECT_URL do Supabase correctos no .env.local
  • lib/prisma.ts criado com o singleton pattern
  • npx prisma db pull funciona sem erros

import { auth } from '@/lib/auth' import { NextResponse } from 'next/server' export default auth((req) => { const { nextUrl, auth: session } = req const isLoggedIn = !!session?.user const isPublicRoute = ['/', '/login'].includes(nextUrl.pathname) const isStaffRoute = nextUrl.pathname.startsWith('/staff') const isApiAuthRoute = nextUrl.pathname.startsWith('/api/auth') // Permitir rotas públicas e auth API sempre if (isApiAuthRoute || isPublicRoute) return NextResponse.next() // Redirecionar para login se não autenticado if (!isLoggedIn) { return NextResponse.redirect(new URL('/login', nextUrl)) } // Proteger rotas /staff — apenas STAFF e ADMIN if (isStaffRoute) { const role = session?.user?.role if (role !== 'STAFF' && role !== 'ADMIN') { return NextResponse.redirect(new URL('/dashboard', nextUrl)) } } return NextResponse.next() }) export const config = { matcher: ['/((?!_next/static|_next/image|favicon.ico).*)'], }
